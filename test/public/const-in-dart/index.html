    Const In Dart

    Const in Dart#

    About#

    const is a keyword in Dart used to define compile-time constants. These constants are immutable and their values are fixed at compile time. The const keyword ensures that the values are determined during compilation, which can optimize performance and ensure immutability.

    Use#

    • Compile-Time Constants: Declaring variables with const means their values are known and fixed at compile time. This makes them immutable and helps optimize performance.

    • Immutable Objects: When used with classes, const constructors create immutable instances. These instances cannot be modified after creation.

    • Constant Expressions: const can be used with expressions that are fully evaluated at compile time. This ensures that the values remain constant throughout the execution of the program.

    Topics#

    • Constant Variables: Variables declared with const are compile-time constants and cannot be reassigned.

    • Const Constructors: Classes can have const constructors to create immutable objects that are initialized at compile time.

    • Compile-Time Evaluation: const helps in evaluating expressions at compile time, which can improve application performance.

    • Using const with Collections: const can be used with collections like lists and maps to create immutable collections that are evaluated at compile time.

    Examples#

    Constant Variables# 

    const int maxUsers = 100;
const String appName = 'MyApp';

void main() {
  print(maxUsers); // Outputs: 100
  print(appName);  // Outputs: MyApp
}

    Explanation: maxUsers and appName are constants whose values are fixed at compile time and cannot be changed.

    Const Constructors# 

    class Point {
  final int x;
  final int y;

  const Point(this.x, this.y); // Const constructor
}

void main() {
  const p1 = Point(1, 2);
  const p2 = Point(1, 2);

  print(identical(p1, p2)); // Outputs: true
}

    Explanation:#

    The Point class uses a const constructor, allowing instances to be created at compile time. p1 and p2 are identical because they are compile-time constants.

    Const with Collections# 

    void main() {
  const List<int> numbers = [1, 2, 3, 4, 5];
  const Map<String, int> ages = {
    'Alice': 30,
    'Bob': 25,
  };

  print(numbers); // Outputs: [1, 2, 3, 4, 5]
  print(ages);    // Outputs: {Alice: 30, Bob: 25}
}

    Explanation: numbers and ages are immutable collections created with const. They cannot be altered after creation.

    Overall#

    The const keyword in Dart defines compile-time constants that are immutable and optimized for performance. It applies to variables, constructors, and expressions known at compile time, ensuring reliability and efficiency in Dart applications.

    import 'dart:async';

// Define a simple class to represent a user
class User {
  final String name;
  final int age;

  const User(this.name, this.age);
}

// Function to simulate fetching user data
Future<User> fetchUser() async {
  // Simulate a network delay
  await Future.delayed(Duration(seconds: 2));
  // Return a constant user object
  return const User('Alice', 30);
}

void main() async {
  // Use const to define immutable messages
  const String welcomeMessage = 'Welcome to the User Info App!';
  const String loadingMessage = 'Loading user data...';

  // Print the welcome message
  print(welcomeMessage);

  // Print the loading message
  print(loadingMessage);

  // Fetch the user data
  User user = await fetchUser();

  // Print user information
  print('User Information:');
  print('Name: ${user.name}');
  print('Age: ${user.age}');
}
